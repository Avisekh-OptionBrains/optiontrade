Epicrise Trading Platform API

Complete API reference for the Epicrise trading system

Back to Dashboard

Overview

The Epicrise Trading Platform API provides programmatic access to trading operations, user management, and real-time market data.

Base URL

http://localhost:3000/api

Features

  • Real-time order placement and management
  • Multi-broker support (Angel One, Motilal Oswal)
  • WebSocket real-time updates
  • Comprehensive user management
  • Advanced analytics and reporting

Trading Endpoints

POST /Epicrise/Telegram

Process trading signals from Telegram messages

Request Body
{
  "message": "ER Buy SBIN at 855.05 with Stop Loss at 850.00"
}
Response
{
  "message": "Message processed and orders placed successfully",
  "parsedData": {
    "symbol": "SBIN",
    "transactionType": "Buy",
    "price": 855.05,
    "stopLoss": 850.00
  }
}
POST /Epicrise/AngelOne

Place orders directly to Angel One broker

Request Body
{
  "messageText": "ER Buy SBIN at 855.05 with Stop Loss at 850.00"
}
POST /Epicrise/MotilalOswal

Place orders directly to Motilal Oswal broker

Request Body
{
  "messageText": "ER Buy SBIN at 855.05 with Stop Loss at 850.00"
}

Dashboard API

GET /api/enhanced-dashboard/data

Get comprehensive dashboard data including statistics, recent orders, and charts data

Response
{
  "statistics": {
    "total": 150,
    "successful": 142,
    "failed": 8,
    "successRate": 94.67,
    "totalMessages": 200,
    "totalClients": 25
  },
  "recentOrders": [...],
  "hourlyData": [...],
  "topSymbols": [...]
}
GET /api/enhanced-dashboard/stats/realtime

Get real-time statistics for the last 5 minutes

GET /api/enhanced-dashboard/export/{format}

Export data in JSON or CSV format

Parameters
Parameter Type Description
format string Export format: 'json' or 'csv'
startDate string Start date for data export (optional)
endDate string End date for data export (optional)

WebSocket

Real-time updates via WebSocket connection

Connection URL

ws://localhost:3000/ws

Message Types

  • new_order - New order placed
  • order_update - Order status updated
  • new_message - New trading message received
  • system_alert - System alerts and notifications

Example Message

{
  "type": "new_order",
  "data": {
    "clientName": "John Doe",
    "broker": "ANGEL",
    "symbol": "SBIN",
    "transactionType": "BUY",
    "price": 855.05,
    "quantity": 100,
    "status": "SUCCESS"
  },
  "timestamp": "2024-01-15T10:30:00.000Z"
}

Error Codes

Code Description Solution
400 Bad Request - Invalid message format Check message format: 'ER Buy/Sell SYMBOL at PRICE with Stop Loss at STOPLOSS'
404 Symbol not found in database Ensure the symbol exists in the trading database
404 No clients found Add clients using the user management interface
500 Internal Server Error Check server logs for detailed error information
